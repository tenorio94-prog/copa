Os outros são adiados até que o hábito diário esteja comprovado.

● 2

Quick Read — O verdadeiro MVP

A menor unidade de valor do Copa Pulse. 15 segundos. 3 bullets. Zero distração. O usuário abre, lê, entende, sai.

🔥 Resumo do dia • 15 segundos
Marrocos faz história como primeira seleção africana na semifinal
🌍 Marrocos 1×0 Portugal — Primeiro africano a chegar na semi em 92 anos de Copa
🇫🇷 França 2×1 Inglaterra — Atual campeã segue viva. Inglaterra perde pênalti no fim
⚽ Amanhã: Argentina × Croácia na semifinal
📖 15s de leitura🎧 90s de áudio

📋 Especificação do Quick Read

interface QuickRead {
  // 1 linha, 10-14 palavras
  hero: string

  // 3 bullets. Cada um: 10-15 palavras
  bullets: [string, string, string]

  // Tempo de leitura em segundos (sempre ≤ 20)
  readingTime: number

  // Metadados
  stories: number   // quantos jogos ocorreram
  audioAvailable: boolean
}

Geração: 3 linhas a partir do top 3 EditorialStory. Sempre 3 bullets. Se só há 1 história, as outras 2 viram contexto adicional.

● 3

Homepage do Hábito Diário

Se o usuário tem 30 segundos, a ordem precisa ser cirúrgica. Cada seção ou entrega valor imediato ou é removida.

1. Quick Read (15s). Hero + 3 bullets. O usuário vê isso PRIMEIRO, antes do hero card atual. Se ele quiser mais profundidade, o Hero Story está logo abaixo.
2. Hero Story (5s). Headline editorial + tag + why it matters. O Quick Read já deu o resumo. O Hero Story dá a profundidade editorial para quem quer mais.
3. Jogos de hoje (5s). Ao vivo + próximos. Ação imediata. "O que posso assistir agora?"
4. Tabela (5s). Classificação. Apenas um grupo, o mais relevante. Seletor de grupos secundário.

Ordem final — 30 segundos: 15s Quick Read → 5s Hero Story → 5s Jogos → 5s Tabela

● 4

Eliminar Complexidade

🗑️ Remover do MVP

  • Newsletter: Exige email service, duplicação de conteúdo diário. Adicionar quando houver audiência.
  • Push: Exige service worker, permissão do usuário, servidor de push. Adicionar quando houver retenção.
  • Social Cards (OG Image): Exige @vercel/og, geração server-side. Adicionar quando houver compartilhamento orgânico.
  • Hashtags: Zero valor para o usuário. Só para SEO/marketing.

✅ Manter no MVP

  • Quick Read: 3 linhas de template. Zero infraestrutura. Máximo valor.
  • Homepage Hero: Já existe. Apenas integrar ao StoryPackage.
  • Share Text: 1 linha de template. Compartilhar é viralidade gratuita.
  • Daily Recap: Composto automaticamente das stories. Sem custo adicional.

⏰ Adiar

  • Audio: 90s de áudio é diferencial mas não cria hábito diário. Implementar quando houver equipe ou orçamento.
  • Subject Lines: Newsletter adiada.
  • Recap visual: Wrapped-style, pós-Copa.
● 5

Loop de Hábito — Por que voltar amanhã?

Gatilho 1 — Continuidade narrativa (força: alta)

"Marrocos venceu Portugal. Amanhã enfrenta a França na semi." O usuário volta para saber o desfecho de uma história que já acompanha. A Copa tem uma narrativa natural de começo, meio e fim. Nosso trabalho é não quebrar essa corrente.

Gatilho 2 — Medo de perder contexto (força: alta)

"Se eu não abrir o Copa Pulse hoje, não vou entender o que está acontecendo no torneio." O Quick Read resolve isso em 15 segundos. O custo de "se manter atualizado" é baixíssimo — e o custo de "ficar por fora" é altíssimo socialmente.

Gatilho 3 — Identidade nacional (força: média)

"Brasil jogou hoje." Torcedor quer saber. Mas isso só funciona nos dias de jogo do seu time. Não sustenta hábito diário para todos os usuários.

Gatilho 4 — Progressão do torneio (força: média)

"Estamos nas quartas. Quem já caiu? Quem ainda está vivo?" O usuário volta para ver o "tabuleiro" atualizado. A tabela e o mata-mata são o tabuleiro.

Gatilho 5 — Hábito fixo (força: baixa no início)

"Toda manhã eu vejo o Copa Pulse." Só funciona depois de 2+ semanas de uso consistente. Não é um ponto de partida, é um ponto de chegada.

Resumo: O melhor gatilho é a continuidade narrativa. A Copa tem começo, meio e fim — o Copa Pulse deve ser o marcador de progresso. "Você acompanhou Marrocos até aqui. Não pare agora." O Quick Read é a ferramenta que torna esse acompanhamento indolor.

● 6

Sistema de Continuidade

O usuário acompanha a Copa por 30+ dias. Cada dia é um capítulo. Precisamos mostrar que o torneio está progredindo sem criar uma timeline infinita.

Proposta: "Onde estamos" — 1 linha no topo da página

📅 Dia 23 da Copa 🏆 Fase de grupos ⚽ 2 jogos hoje 🔥 Maior história: Marrocos na semi

Uma barra no topo. Não precisa de click. Apenas orientação. Gera o "medo de perder contexto".

Proposta: "A jornada continua" — transição entre dias

"Ontem: Marrocos eliminou Portugal 🇲🇦. Hoje: Argentina enfrenta Croácia 🇦🇷🇭🇷." Uma linha conectando o dia anterior ao dia atual. Cria a sensação de que o usuário está acompanhando UMA HISTÓRIA, não notícias soltas.

● 7

StoryBrief — A unidade mínima de valor

interface StoryBrief {
  // 10-14 palavras. Lida sem clique.
  headline: string

  // 12-18 palavras. O que muda no torneio.
  whyItMatters: string

  // 1-3 curtas evidências
  evidence: string[]

  // 1 frase pronta para copiar
  shareText: string

  // Metadados editoriais
  storyType: EditorialStoryType
  tag: string
  confidence: number

  // Tempo estimado de leitura
  readingTime: number  // sempre ≤ 20s
}

StoryBrief é o que o usuário vê nos primeiros 15 segundos. StoryPackage é o que ele vê se quiser mais. A relação: StoryBrief ⊂ StoryPackage — o Brief é um subconjunto do Package, não uma entidade separada.

● 8

Comparação: Valor por Segundo

📋 StoryBrief (15s)

"Marrocos faz história como primeira seleção africana na semi"
• Marrocos 1×0 Portugal — feito inédito em 92 anos de Copa
• França 2×1 Inglaterra — atual campeã segue viva
• Amanhã: Argentina × Croácia

Valor por segundo: ALTO. 3 informações em 15s.

📰 Homepage Hero (30s+)

"Marrocos faz história como primeira seleção africana a chegar à semifinal"
Em 92 anos de Copa, nenhum país africano tinha ido tão longe. Marrocos não tomou um gol sequer no mata-mata. A defesa virou muralha.

Valor por segundo: MÉDIO. Mais contexto, mas o usuário já sabia pelo Brief.

Veredito: StoryBrief gera MAIS valor por segundo que qualquer outro formato. O usuário consome o dia em 15 segundos. Se quiser mais, o Hero Story está a 1 scroll de distância. O Brief não substitui o Hero — ele é a porta de entrada.

● 9

Value Density Score

📊 Como medir "valor por segundo"

Value Density = Informação Útil / Tempo de Consumo

Informação Útil: Número de fatos que o usuário lembra 5 minutos depois de consumir. Não é o que o sistema produziu — é o que o usuário reteve.

Proxy prático para MVP: Número de evidence[] que aparecem na story com maior confidence dividido por 15 segundos. Um Quick Read com 3 evidence em 15 segundos tem density de 0.2 fatos/segundo. Um audio de 90 segundos com 5 evidence tem density de 0.05 fatos/segundo.

O Quick Read é 4x mais denso que o áudio.
