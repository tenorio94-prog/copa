● 1

Roteiro de Teste — 10 Usuários

📋 Setup

Duração: 10-15 minutos por usuário. Remoto ou presencial.
Perfil: 5 homens, 5 mulheres. 25-40 anos. Acompanham Copa mas não são viciados em futebol.

● 3

Operação Mínima — VPS + Cron

🚀 Arquitetura de Operação

Cron
*/5 * * * * → fetch-data.sh
Node script → latest.json
disco → Next.js
ISR revalidate

📄 fetch-data.sh

#!/bin/bash
# /opt/copa-pulse/fetch-data.sh
# Chamado pelo cron a cada 5 minutos

cd /opt/copa-pulse

# 1. Fetch fixtures da API-Football
curl -s -H "x-rapidapi-key: $API_FOOTBALL_KEY" \
  "https://v3.football.api-sports.io/fixtures?live=all" \
  -o /tmp/live-fixtures.json

curl -s -H "x-rapidapi-key: $API_FOOTBALL_KEY" \
  "https://v3.football.api-sports.io/fixtures?date=$(date +%Y-%m-%d)&league=1&season=2026" \
  -o /tmp/date-fixtures.json

# 2. Fetch standings
curl -s -H "x-rapidapi-key: $API_FOOTBALL_KEY" \
  "https://v3.football.api-sports.io/standings?league=1&season=2026" \
  -o /tmp/standings.json

# 3. Processar com Node
node scripts/build-data.mjs

# 4. Mover para o diretório servido pelo Next.js
cp /tmp/latest-brief.json /opt/copa-pulse/public/data/latest.json

⏰ Crontab

# A cada 5 minutos durante a Copa
*/5 * * * * /opt/copa-pulse/fetch-data.sh >> /var/log/copa-pulse.log 2>&1

# Diariamente às 06:00 — gera boletim + LLM
0 6 * * * /opt/copa-pulse/scripts/generate-brief.sh >> /var/log/copa-pulse-daily.log 2>&1

📦 Dependências da VPS

  • Node.js 18+ (npm install no projeto)
  • curl (para API-Football)
  • cron (pré-instalado em Linux)
  • Nginx (reverse proxy para Next.js na porta 3000)
  • ~2GB RAM, 10GB disco — $10-15/mês

● 5

Veredito Final

🎯 O que falta para colocar nas mãos de 10 usuários?

1. Assinar API-Football ($99/ano) ⏱ 1h

Produto não funciona sem dados reais. A chave está no .env.local. Basta pagar e configurar.

2. Configurar cron na VPS ⏱ 2h

fetch-data.sh + scripts/build-data.mjs + crontab. ~30 linhas de script. Documentado no Entregável 3.

3. Rodar o roteiro com 10 usuários ⏱ 1 semana

Recrutar 10 pessoas. 15 minutos cada. Seguir o roteiro do Entregável 1. Coletar métricas.

4. Analisar resultados ⏱ 1 dia

Comparar com thresholds do Entregável 2. Decidir: aprova, aprova com ressalvas, ou volta para Product Audit.

Resumo: Faltam 4 ações concretas. Nenhuma delas é nova funcionalidade. Nenhuma delas é nova tela. Nenhuma delas é nova IA.

O produto editorial está pronto. A operação precisa ser montada. Os usuários precisam ser convidados.

O Copa Pulse pode estar nas mãos dos primeiros 10 usuários em 1 semana de trabalho operacional.
