Habit Loop Engine

Não é push. Não é newsletter. O hábito vem da história. Cada dia é um capítulo que termina com "continua amanhã".

● 1

Por que voltar amanhã?

1

Continuidade narrativa

A história não acabou. Marrocos eliminou Portugal. Amanhã enfrenta a França. O usuário precisa saber o desfecho.

Força: alta
2

Progressão do torneio

A Copa avança. Fase de grupos → oitavas → quartas → semi → final. Cada dia elimina times. O "tabuleiro" muda. O usuário quer ver o novo cenário.

Força: alta
3

Medo de perder contexto

"Se eu não abrir hoje, vou ficar por fora amanhã." O Quick Read torna o custo de se atualizar muito baixo (15s). O custo de ficar por fora é alto socialmente. A assimetria favorece o hábito.

Força: alta
4

Curiosidade sobre desfecho

"Será que Marrocos consegue?" Não é uma notícia. É uma pergunta. O produto deve terminar o dia com uma pergunta, não com uma resposta.

Força: média
5

Acompanhar uma zebra

O usuário se apega a narrativas de underdog. "Quero ver até onde Marrocos vai." Funciona — mas depende de existir uma zebra ativa.

Força: média
6

Acompanhar uma seleção

"Brasil joga hoje." Funciona apenas nos dias de jogo do time do usuário. Não sustenta hábito diário.

Força: baixa
Conclusão: O melhor gatilho não é notificação. É história inacabada. O produto deve terminar cada dia com uma pergunta que só será respondida amanhã. "Marrocos enfrenta a França amanhã. Será que a zebra chega à final?" — isso é mais poderoso que qualquer push.
● 2

Entidade HabitLoop

interface HabitLoop {
  // O gancho que faz o usuário querer voltar
  hook: string

  // A pergunta que será respondida amanhã (ou em breve)
  openQuestion: string

  // O próximo jogo/evento relevante
  nextEvent: {
    matchId: string | null
    label: string       // "Argentina vs Croácia"
    date: string        // 2026-12-13
    importance: number  // 0-100
  } | null

  // A narrativa ativa que o usuário está acompanhando
  activeNarrative: {
    type: string        // "cinderella", "redemption", "dynasty_fall"
    team: string
    daysActive: number
    nextChapter: string // "Marrocos tenta chegar à final"
    status: "ongoing" | "completed" | "failed"
  } | null

  // Força do hábito (0-1)
  retentionScore: number
}
Origem dos dados: EditorialStoryEngine já produz NarrativeArcs. HabitLoop é uma view desses arcs com foco no FUTURO. O arc `cinderella_progression` vira `activeNarrative.type = "cinderella"` com `nextChapter = "Marrocos tenta chegar à final"`. O arc `redemption_journey` vira `activeNarrative.type = "redemption"` com `nextChapter = "Argentina busca completar a redenção"`.

● 4

Followed Narrative

O sistema não precisa de login para "seguir" uma narrativa. As narrativas globais da Copa são poucas e evidentes. O sistema mostra a mais importante. 

interface FollowedNarrative {
  type: string
  team: string
  startedAt: string     // data do primeiro jogo relevante
  status: "ongoing" | "completed" | "failed"

  // História até agora — gerada dos teamJourneys
  journey: string[]

  // O que está em jogo agora
  currentStake: string

  // O que acontece se vencer / perder
  nextIfWin: string
  nextIfLoss: string

  editorialWeight: number
}

// Não há "escolha do usuário". O sistema mostra a narrativa ativa de maior weight.
// O usuário não segue — ele é apresentado. O FollowedNarrative é global.

Princípio: narrativa global, não personalizada. A Copa tem 2-3 narrativas globais por edição (Marrocos 2022, Messi 2022, França bicampeã). O sistema mostra a maior delas. Não precisa de login, perfil, ou preferências para fazer o usuário se importar.

● 5

Métrica: Razões para Voltar

📊 Retention Potential Score

Fórmula: RPS = openQuestions × narrativeDepth × nextEventImportance

openQuestions: Quantas perguntas o dia deixa em aberto (0-3). "Marrocos enfrenta a França amanhã" = 1 pergunta.
narrativeDepth: Dias consecutivos da mesma narrativa (1-30). Quanto mais dias, mais investimento emocional.
nextEventImportance: Importância do próximo jogo (0-1). Final = 1.0. Semifinal = 0.8. Fase de grupos = 0.3.

Exemplo: Marrocos 5 dias de narrativa + semifinal amanhã = 1 × 5 × 0.8 = 4.0 (probabilidade alta de retorno)
Exemplo: Fase de grupos sem zebra + jogo qualquer amanhã = 0 × 2 × 0.3 = 0 (probabilidade baixa)

🎯 Qual elemento mais aumenta a chance de retorno?

Next Chapter. O StoryBrief informa o que aconteceu. A ContinuityBar mostra o progresso. Mas o Next Chapter CRIA a necessidade de voltar: "Marrocos enfrenta a França amanhã. Será que a zebra chega à final?" — Essa pergunta não será respondida hoje. O usuário precisa voltar amanhã.

O StoryBrief é sobre o PASSADO. O Next Chapter é sobre o FUTURO. O hábito mora no futuro.

● 7

O que não fazer

✅ ÚTIL — usar com moderação

Push notifications: Apenas para eventos de weight ≥ 85. Máximo 1/dia. "Marrocos na semi" sim. "Brasil venceu grupos" não.

🟡 NEUTRO — não ajuda nem atrapalha

Newsletter: Bom para retenção a longo prazo, mas exige email. Não é urgente.
Gamificação leve: "Você está acompanhando há X dias." Pode ajudar, mas não substitui Next Chapter.

❌ PREJUDICIAL — não fazer no MVP

Streaks: "Você perdeu um dia!" — Culpa não cria hábito, cria ansiedade.
Badges / Conquistas: Parece exercício de startup, não produto editorial.
Ranking de usuários: Zero relação com o valor do produto.
Comentários: Exige moderação. Desvia do foco editorial.
Personalização / Login: Barreira de entrada. O produto deve funcionar SEM cadastro.

● 8

Roadmap de Retenção

🎯 O menor conjunto que aumenta retenção

1
Next Chapter na homepage

Componente que mostra o próximo capítulo da história ativa. Abaixo do Hero Story. ~20 linhas. Geração automática a partir do NarrativeArc ativo de maior weight.

2
HabitLoop entity + buildHabitLoop()

Entidade que extrai o "gancho de retorno" do EditorialStoryEngine. ~30 linhas. Sem novos dados.

3
Push notifications (restrito)

Service worker + regras de disparo. Apenas eventos weight ≥ 85. ~50 linhas. Exige HTTPS (Vercel).

Resumo: O Next Chapter é a funcionalidade de retenção mais importante. Custa ~20 linhas. Não exige novos dados. Não exige permissão do usuário. E responde diretamente à pergunta "por que eu voltaria amanhã?" — porque a história não acabou.
