Story Package System

Uma história. Sete formatos. Zero duplicação editorial.

● 1

Entidade StoryPackage

interface StoryPackage {
  // Referência à história fonte
  story: EditorialStory

  // Formato principal — homepage
  homepage: {
    headline: string       // 8-15 palavras
    summary: string        // 2-3 frases
    whyItMatters: string   // 1 frase
    tag: string
    storyType: string
    evidence: string[]
  }

  // Áudio — 90 segundos
  audio: {
    script: string         // roteiro completo
    duration: number       // segundos
    chapters: { label, start }[]
    hook: string
    closing: string
  } | null

  // Newsletter — snippet matinal
  newsletter: {
    subject: string        // 5-8 palavras
    preview: string        // 1 frase
    body: string           // 2-3 parágrafos
  } | null

  // Redes sociais
  social: {
    cardText: string       // 150-200 caracteres
    cardSubtext: string    // 1 linha
    ogImage: string        // URL da imagem gerada
    hashtags: string[]
  } | null

  // Share — texto para copiar
  share: {
    text: string           // 1 frase, pronta para copiar
    link: string
  }

  // Push notification
  push: {
    title: string          // 5-8 palavras
    body: string           // 1 frase
    urgency: "normal" | "high"
  } | null

  // Daily recap — resumo do dia
  recap: {
    heroHeadline: string
    secondaryHeadline: string | null
    emergingHeadline: string | null
    storiesCount: number
    matchesToday: number
    audioDuration: number
  }
}
● 2

Formatos de Saída

1. Homepage Hero

Tamanho: 8-15 palavras headline • 2-3 frases summary • Prioridade: máxima
Já existe. É o EditorialStory atual. Precisa de headline editorial, não factual. "Marrocos faz história como primeira seleção africana na semi" — não "Marrocos 1×0 Portugal".
📐 Já implementado. Apenas integrar ao StoryPackage.

2. Audio Script

Tamanho: 90 segundos • 150-200 palavras • Tom: conversacional
7 blocos: Hook → Context → Result + Why → Historical Weight → Secondary → Emerging → Closing. O Script Generator (Sprint 2.2) já define isso.
📐 Já especificado. Integrar ao StoryPackage como campo audio.script.

3. Newsletter Snippet

Tamanho: 3-5 parágrafos • 150-200 palavras • Tom: editorial premium
O resumo matinal que o usuário recebe por email. Deve funcionar sozinho (quem abre o email não abre o site). Incluir: hero + secondary + why matters + link para áudio.
📐 Novo. Precisa de template e geração.

4. Social Card

Tamanho: 150-200 caracteres • 1 imagem • Tom: compartilhável
O card que o usuário compartilha no WhatsApp, X, Instagram. Deve ser informativo sozinho + fazer quem vê querer abrir o link. O fato histórico ou o dado surpreendente é o gancho.
📐 Novo. Precisa de design e geração de imagem.

5. Share Text

Tamanho: 1 frase • 15-25 palavras • Tom: orgulho de compartilhar
Texto pré-escrito para copiar e colar. O usuário não precisa pensar no que escrever. "A Argentina é campeã mundial após 36 anos. Messi finalmente tem a Copa que merecia. [link]"
📐 Novo. Geração simples baseada no story type.

6. Push Notification

Tamanho: 5-8 palavras título • 10-15 palavras body • Tom: urgência seletiva
Apenas para histórias de alto impacto. Não para resultados normais. "Brasil eliminado da Copa" — sim. "Brasil vence fase de grupos" — não.
📐 Novo. Precisa de regras editoriais de quando enviar.

7. Daily Recap

Tamanho: 3 seções • 50 palavras total • Tom: resumo executivo
O "fim do dia". Hero + Secondary + Emerging em formato de lista. Ideal para: final da noite, resumo antes de dormir, compartilhar com amigos que perderam o dia.
📐 Novo. Composto automaticamente das 3 stories selecionadas.
● 3

Regras Editoriais por Canal

🏠 Homepage

  • Headline editorial, não factual
  • Why It Matters sempre visível
  • Evidence list como segunda camada
  • Tag e storyType como metadata

🎙️ Audio

  • Hook antes do resultado
  • 1 fato histórico no meio
  • Closing thought memorável
  • 90 segundos, ritmo conversacional

📧 Newsletter

  • Subject: hook + time
  • Preview: why it matters
  • Corpo: hero → secondary → emerging
  • CTA: "Ouça o resumo em 90s"

📱 Social Card

  • Headline grande (18-22px)
  • 1 fato surpreendente como subtitle
  • Tag no topo (🔥 Redenção, 🌍 Marco)
  • Logo Copa Pulse no rodapé
  • Formato: 1080×1080 ou 1080×1920

🔔 Push Notification

  • Apenas editorialWeight ≥ 85
  • Nunca para grupos
  • Nunca para resultados esperados
  • Máximo 1 push por dia
  • Silencioso entre 22h e 8h

📋 Share Text

  • 1 frase que funciona isolada
  • Sempre terminar com link
  • Nunca pedir "compartilhe"
  • Usar emoji no início

Social Cards

O que faz alguém compartilhar um card? Dado surpreendente + design premium + orgulho de associar a marca.

✅ O que faz alguém compartilhar

  • Surpresa: "36 anos" é mais compartilhável que "Argentina campeã"
  • Identidade: Torcedor compartilha por orgulho nacional
  • Utilidade: "Resumo em 90s" é um bônus que justifica o clique
  • Status: Saber um fato histórico que outros não sabem

❌ O que NÃO faz compartilhar

  • Placar puro ("Argentina 3×3 França")
  • Headline genérica ("Grande jogo hoje")
  • Sem contexto histórico ("Argentina venceu")
  • Design poluído ou genérico
● 6

Push Notifications

✅ Quando enviar

  • Eliminação de favorito: "Brasil eliminado da Copa nos pênaltis"
  • Marco histórico: "Marrocos é o primeiro africano a chegar à semifinal"
  • Final definida: "Argentina e França decidem o título no domingo"
  • Resultado histórico: "Arábia Saudita vence Argentina na maior zebra da história"
  • Lesão de lendário: "Neymar está fora da Copa por lesão"

Regra: apenas quando editorialWeight ≥ 85 E stage não é fase de grupos.

❌ Quando NÃO enviar

  • Vitória de favorito: "Brasil vence fase de grupos"
  • Jogo normal: "França 2×0 Dinamarca"
  • Classificação esperada: "Alemanha vence e segue viva"
  • Múltiplos pushes no mesmo dia: Máximo 1 push diário
  • Período noturno: Entre 22h e 8h, silencioso

● 10

Roadmap

✅ MVP — StoryPackage Core

Homepage Hero + Share Text + Daily Recap. Os 3 formatos que não exigem novos canais de distribuição. Homepage já existe. Share text é gerado com template de 1 linha. Daily recap é composto das 3 stories já selecionadas.

homepage share text daily recap

⏰ Sprint 3.1 — Distribuição

Social Cards (geração de OG image + template de card) + Audio Script integration. Social cards exigem geração de imagem server-side (@vercel/og). Audio usa o Script Generator da Sprint 2.2.

social cards og images audio integration

⏰ Sprint 3.2 — Canais Externos

Newsletter (setup de email, template diário) + Push Notifications (service worker, regras editoriais). Ambos exigem canais de distribuição que o MVP não tem.

newsletter push

📅 Pós-Copa

Wrapped (resumo do torneio), App Nativo (notificações nativas), Recomendação personalizada.

🎯 MVP — O menor StoryPackage possível

3 formatos, 0 canais novos. Homepage (já existe) + Share Text (template) + Daily Recap (composto). Nenhuma infraestrutura nova. Nenhum canal externo. O MVP do StoryPackage é simplesmente estruturar os dados que o sistema já produz em um formato que o frontend pode consumir de forma unificada.

O que muda no frontend: O Hero da homepage passa a consumir storyPackage.homepage em vez de editorialStory.headline. A diferença? storyPackage já vem com todos os formatos prontos.

Ganho esperado: Compartilhamento (+). Retenção (++). Consistência entre canais (+++). Uma história, uma fonte da verdade, N versões.
